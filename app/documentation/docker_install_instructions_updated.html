Instrucciones de Instalación de Docker en Ubuntu

1. Actualizar el sistema

sudo apt update
sudo apt upgrade

2. Instalar dependencias necesarias

sudo apt install apt-transport-https ca-certificates curl software-properties-common

3. Agregar el repositorio de Docker

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

4. Instalar Docker

sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io

5. Verificar la instalación de Docker

sudo docker --version

6. Permitir a tu usuario ejecutar Docker (opcional)

sudo usermod -aG docker $USER

Después de esto, cierra la sesión y vuelve a iniciarla para que los cambios tomen efecto.

7. Instalar Docker Desktop en Ubuntu

Descargar Docker Desktop desde el siguiente enlace:

Descargar Docker Desktop

sudo apt install ./docker-desktop--.deb

8. Iniciar Docker Desktop

docker-desktop

9. Verificar Docker Desktop

docker --version
docker-compose --version

10. Instalar Docker Compose (si es necesario)

Si Docker Compose no se instaló junto con Docker Desktop o si prefieres instalarlo manualmente, puedes hacerlo de la siguiente manera:

sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.3/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

Después, verifica la instalación con:

docker-compose --version

11. Recomendación

No es necesario instalar Docker Compose manualmente si ya tienes Docker Desktop instalado. Docker Desktop incluye Docker Compose automáticamente.

Si instalas Docker Compose manualmente, asegúrate de que las rutas no se crucen para evitar problemas. Puedes verificar qué versión estás utilizando ejecutando:

which docker-compose
docker-compose --version
